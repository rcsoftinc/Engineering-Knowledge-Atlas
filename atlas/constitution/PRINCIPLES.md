# Principios

1. Todo concepto existe porque resuelve un problema.
2. Ningún concepto existe aislado.
3. Las relaciones importan tanto como las definiciones.
4. Los conceptos preceden a las herramientas.
5. La arquitectura precede a la implementación.
6. La claridad precede a la exhaustividad.
7. La calidad precede a la velocidad.
8. La coherencia precede al volumen.
9. El contexto real precede a la abstracción vacía.
10. La seguridad forma parte del diseño.
11. La operación forma parte del sistema.
12. La documentación forma parte del producto.
13. Las decisiones importantes deben ser trazables.
14. El repositorio es la fuente de verdad.
15. Las IA colaboran bajo reglas explícitas.
16. La base estable y la operación viva permanecen separadas.
17. Toda release produce valor verificable.
18. Todo artefacto debe poder mantenerse.
19. Lo automatizable podrá automatizarse después de comprenderse.
20. El Atlas debe sobrevivir a las tecnologías que documenta.
