# Architectural Decision Records

## ADR-0001 — Español como idioma principal
Status: Accepted

El contenido original se redactará en español. La arquitectura permitirá traducciones futuras.

## ADR-0002 — Markdown como fuente canónica inicial
Status: Accepted

Markdown es legible, versionable, portable y compatible con GitHub e IA.

## ADR-0003 — Git como memoria histórica
Status: Accepted

Git conserva evolución, decisiones, releases y autoría.

## ADR-0004 — El Atlas es AI-native
Status: Accepted

Los documentos deben ser comprensibles para humanos y estructurados para IA.

## ADR-0005 — Knowledge Objects
Status: Accepted

Conceptos, flujos, comparaciones, proyectos, rutas y rabbit holes son objetos con metadatos y relaciones.

## ADR-0006 — Coherencia como medida de release
Status: Accepted

Una release se mide por la coherencia de sus artefactos, no por la cantidad de archivos.

## ADR-0007 — Congelación de alcance en Producción
Status: Accepted

Cuando comienza Producción, las nuevas ideas se difieren salvo errores críticos.

## ADR-0008 — Separación constitucional y operativa
Status: Accepted

La base permanente cambia raramente. Estado, backlog y checklists cambian con frecuencia.

## ADR-0009 — Protocolo humano–IA
Status: Accepted

La aprobación explícita obliga a pasar de propuesta a ejecución.

## ADR-0010 — Un único repositorio en esta etapa
Status: Accepted

Conocimiento, gobernanza y herramientas iniciales vivirán juntos hasta que exista una razón práctica para separarlos.
